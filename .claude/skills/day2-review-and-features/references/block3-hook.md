## EXPLAIN

### Hook이란?

Hook = 특정 이벤트가 발생하면 **자동으로 실행**되는 셸 스크립트.

### 핵심 개념: 결정론적 vs 확률적

| | AI (확률적) | Hook (결정론적) |
|--|------------|----------------|
| 실행 방식 | "해줘"라고 부탁 | 이벤트 발생 시 자동 |
| 신뢰도 | 대부분 함 (가끔 까먹음) | 100% 실행 |
| 예시 | "파일 저장 후 포맷팅 해줘" | 파일 저장 이벤트 → prettier 자동 실행 |

### 비유: 자동 체크리스트

업무에서 "보고서 작성 후 반드시 팀장에게 공유"라는 규칙이 있다고 하자:
- AI 방식: Claude에게 "보고서 쓰고 나면 공유해줘"라고 매번 부탁 → 가끔 까먹음
- Hook 방식: "보고서 파일 생성 이벤트" → 자동으로 공유 스크립트 실행 → 100% 실행

### 이벤트 종류

| 이벤트 | 시점 | 예시 |
|--------|------|------|
| PreToolUse | 도구 실행 **전** | 파일 수정 전 백업 |
| PostToolUse | 도구 실행 **후** | 파일 저장 후 포맷팅 |
| Stop | 응답 완료 시 | 완료 시간 기록 |

```
사용자 요청
    │
    ▼
┌─ PreToolUse Hook ─┐
│ (도구 실행 전)      │ ← 검증, 백업 등
└────────┬──────────┘
         ▼
    [도구 실행]
         │
         ▼
┌─ PostToolUse Hook ┐
│ (도구 실행 후)      │ ← 포맷팅, 알림 등
└────────┬──────────┘
         ▼
    [응답 생성]
         │
         ▼
┌─ Stop Hook ───────┐
│ (응답 완료 시)      │ ← 시간 기록, 요약 등
└───────────────────┘
```

## EXECUTE

참가자에게 아래 2단계를 순서대로 Claude Code에 입력하라고 안내한다.

### 1단계: Hook 이해하기

Claude에게 아래와 같이 입력한다:

```
Hook이 뭔지 예시와 함께 설명해줘. 내 업무에서 쓸 수 있는 Hook 아이디어 3개도 제안해줘
```

> Claude가 Hook의 개념과 함께, 패션 플랫폼 MD 업무에 맞는 Hook 아이디어를 제안해줄 것이다.

### 2단계: 직접 Hook 추가하기

Claude에게 아래와 같이 입력한다:

```
Stop Hook을 추가해줘. 응답이 끝나면 현재 시간을 터미널에 출력하도록
```

> Claude가 `.claude/settings.local.json`에 Stop 이벤트 Hook을 추가해줄 것이다.
> 설정 후 다음 응답부터 응답이 끝날 때마다 `[완료] 14:32:05` 같은 시간이 터미널에 표시된다.
> Hook이 실제로 동작하는지 아무 질문이나 해서 확인해보자.

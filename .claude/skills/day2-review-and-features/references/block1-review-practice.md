# Block 1: 데이터 실습 복습

> 어제는 기존 데이터를 정리했다면, 오늘은 실무 시나리오: 신규 데이터 병합.

## EXPLAIN

### 실무 상황 설명

패션 플랫폼 MD의 실제 업무 흐름:
- 매주 월요일, 신규 입고 상품 데이터가 들어온다
- 기존 상품 데이터(products.csv)에 신규 데이터(products_new.csv)를 합쳐야 한다
- 같은 SKU(상품 코드)가 있으면 신규 데이터로 업데이트해야 한다
- 병합 후 데이터가 정확한지 검증해야 한다

이 과정을 Claude에게 시켜본다.

### 오늘의 시나리오

"이번 주 신규 입고 상품이 추가되었다. products_new.csv를 기존 products.csv에 병합하고, 중복 SKU를 처리하고, 재고를 업데이트하라."

핵심 포인트:
- 어제는 **하나의 파일 정리** → 오늘은 **두 파일 병합**
- 실무에서 매주 반복되는 작업 → 나중에 스킬로 만들 수 있다

## EXECUTE

참가자에게 아래 3단계를 순서대로 Claude Code에 입력하라고 안내한다.

### 1단계: 신규 데이터 생성

Claude에게 아래와 같이 입력한다:

```
신규 입고 상품 데이터 products_new.csv를 15개 행으로 만들어줘. 기존 products.csv와 같은 컬럼인데, SKU 3개는 기존과 중복되게 해줘
```

> Claude가 기존 products.csv를 읽고, 같은 컬럼 구조로 products_new.csv를 생성한다.
> 15개 행 중 3개는 기존 SKU와 동일하고, 12개는 신규 상품이다.

### 2단계: 데이터 병합

```
products.csv와 products_new.csv를 병합해줘. 중복 SKU는 신규 데이터로 업데이트하고, 병합 결과를 products_merged.csv로 저장해줘
```

> Claude가 두 파일을 읽고 SKU 기준으로 병합한다.
> 중복 SKU는 products_new.csv의 데이터(최신)로 덮어쓴다.

### 3단계: 병합 결과 검증

```
병합 결과를 검증해줘. 총 행 수, 중복 처리 결과, 누락 데이터를 확인해줘
```

> Claude가 병합된 데이터의 정합성을 체크한다.
> 기대 결과: 기존 행 수 + 신규 12개 = 총 행 수 (중복 3개는 업데이트)
